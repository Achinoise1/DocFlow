# 实现计划：文件列表与任务列表拆分

**基于**：PRD_文件列表与任务列表拆分.md  
**版本**：V0.2.1  
**日期**：2026-03-30

---

## 1. 总体实现思路

本次重构以「最小化破坏、分步抽离」为原则，不重写整个主窗口，而是通过以下三步完成：

```
Step 1：瘦身现有组件
    FileListItem（drop_zone.py） → 剥离进度条和状态标签
    FileListWidget               → 剥离所有任务相关方法和字段

Step 2：新建任务组件
    TaskListItem  → 单条任务项（含状态、进度、重试按钮）
    TaskListWidget → 任务列表容器（含清除已完成按钮）

Step 3：重构主窗口
    布局改为 QSplitter 左右分栏
    数据流：开始转换 → 文件移出 FileListWidget → 任务加入 TaskListWidget
    任务回调改为驱动 TaskListWidget
    Tab 切换逻辑改为清空文件列表（含确认弹窗）
    取消全部逻辑改为驱动 TaskListWidget 标记已取消
```

整个过程不涉及 `core/` 层改动（`TaskManager` 已保存完整任务参数，可直接复用）。

## 2. 功能模块拆解与职责

### 2.1 `ui/widgets/drop_zone.py` — `FileListItem` 简化

**目标**：`FileListItem` 只作为文件暂存行的展示，不再承载任务状态。

**移除**：
- `self.status_label`（状态标签）
- `self.progress_bar`（进度条）
- `set_waiting()`、`set_status()`、`set_progress()` 等任务方法

**保留**：
- 文件图标、文件名、文件大小
- `remove_clicked` 信号
- `×` 删除按钮

**最终布局**（横向）：
```
[图标] [文件名 / 大小]                    [× 删除]
```

---

### 2.2 `ui/widgets/file_list_widget.py` — `FileListWidget` 简化

**目标**：回归纯「文件暂存」容器，不感知任务。

**移除的字段**：
- `_file_items` 中的 `task_id`、`done` 字段（仅保留 `widget`）

**移除的方法**：
- `pending_paths`（属性）
- `task_id_map`（属性）
- `set_task_id()`
- `mark_skipped()`
- `reset_all_tasks()`
- `update_task_state()`

**保留的公共接口**：
- `file_paths` — 当前所有文件路径列表
- `add_files(file_paths, accepted_exts)` → `list[str]`（rejected）
- `remove_file(file_path)`
- `clear_files()`
- `collect_files_from_dir(dir_path, accepted_exts)`
- Signal：`file_count_changed(int)`、`file_removed(str)`

### 2.3 `ui/widgets/task_list_widget.py` — 新增（核心新组件）

#### 2.3.1 数据结构：任务快照

每条任务在内存中保存一个快照字典，供重试时读取：

```python
TaskSnapshot = {
    'task_id':         str,          # 当前任务ID
    'title':           str,          # 显示标题（文件名或合并标题）
    'conversion_type': str,          # 转换类型 ID
    'input_path':      str | None,   # 单文件任务的源文件路径
    'image_paths':     list | None,  # 批量合并任务的图片路径列表
    'output_dir':      str | None,   # 输出目录
    'output_path':     str | None,   # 批量任务的输出文件路径（重试时重新询问）
    'options':         dict,         # 额外选项（dpi、format 等）
    'is_batch':        bool,         # 是否为批量合并任务
}
```

#### 2.3.2 `TaskListItem`（QWidget）

**显示内容**（纵向两行 + 横向布局）：

```
行1: [状态色块] [标题]                          [进度条]  [状态标签]
行2: [转换类型]  [错误信息 / 输出文件名（可点击）]           [重试/重新开始]
```

**状态 → 视觉映射**：

| 状态 | 颜色标记 | 操作按钮 |
|------|----------|----------|
| 等待中 | 灰色 | — |
| 转换中 | 蓝色 | — |
| 已完成 | 绿色 | — |
| 失败 | 红色 | 「重试」 |
| 已取消 | 橙色 | 「重新开始」 |

**对外信号**：
- `retry_requested(task_id: str)` — 用户点击重试/重新开始

**公共方法**：
- `set_state(state: str, percent: int, message: str, output_path: str)`

#### 2.3.3 `TaskListWidget`（QWidget）

**布局**：
```
[任务列表 (0)]  [清除已完成]    ← 顶部操作栏
─────────────────────────────
[TaskListItem]
[TaskListItem]
...
[stretch]
```

**对外公共接口**：

```python
# 添加任务（开始转换时由 main_window 调用）
add_task(snapshot: dict) -> None

# 任务状态更新（TaskManager 回调后由 main_window 转发）
on_task_started(task_id: str) -> None
on_task_progress(task_id: str, percent: int) -> None
on_task_finished(task_id: str, success: bool, message: str, output_path: str) -> None

# 批量标记为已取消（取消全部时调用）
cancel_all_pending() -> None

# 清除已完成条目
clear_completed() -> None
```

**对外信号**：
- `retry_requested(task_id: str)` — 冒泡自 `TaskListItem`，由 main_window 处理重试逻辑

**内部状态**：
- `_items: dict[task_id, TaskListItem]` — task_id 到 item widget 的映射
- `_snapshots: dict[task_id, dict]` — task_id 到任务快照的映射

> `_snapshots` 由 `TaskListWidget` 自己维护，重试时 `main_window` 通过 `get_snapshot(task_id)` 取出，发起重新提交。

### 2.4 `ui/main_window.py` — 主窗口重构

#### 2.4.1 布局变更

**现在**（垂直堆叠）：
```
标题栏
Tab（转换类型）
拖拽区
快捷按钮行
文件列表
操作按钮行（输出目录 + 开始转换）
状态栏
```

**目标**（QSplitter 左右分栏）：
```
标题栏
Tab（转换类型）                         ← 全宽
─────────────────────────────────────
左侧 QWidget          │ 右侧 QWidget
  拖拽区              │   任务列表标题 + 清除已完成
  快捷按钮行          │   TaskListWidget（可滚动）
  文件列表标题 + 清空  │
  FileListWidget      │
─────────────────────────────────────
操作按钮行（输出目录 + 开始转换 + 取消）  ← 全宽
状态栏                                  ← 全宽
```

**QSplitter 配置**：
- 方向：`Qt.Horizontal`
- 初始尺寸比：`[1, 1]`（`setSizes([450, 450])`）
- 单侧最小宽度：200px（`setMinimumWidth(200)`）

#### 2.4.2 数据流变更

**`_start_conversion()` 改动**：
```python
# 旧：提交任务、在 FileListWidget 上打标记、靠 done 字段判断 pending
# 新：
for path in pending_paths:
    task_id = uuid4()[:8]
    snapshot = build_snapshot(task_id, path, ...)
    self.task_list_widget.add_task(snapshot)   # 先加入任务列表
    self.file_list_widget.remove_file(path)    # 再从文件列表移除
    self.task_manager.submit_task(task_id, ...)
```

**任务回调改动**：
```python
def _on_task_started(self, task_id):
    self.task_list_widget.on_task_started(task_id)

def _on_task_progress(self, task_id, percent):
    self.task_list_widget.on_task_progress(task_id, percent)

def _on_task_finished(self, task_id, success, message, output_path):
    self.task_list_widget.on_task_finished(task_id, success, message, output_path)
    # 自动打开逻辑保留，从 output_path 收集
```

**重试逻辑（新增）**：
```python
def _on_retry_requested(self, task_id: str):
    snapshot = self.task_list_widget.get_snapshot(task_id)
    if not snapshot:
        return
    if snapshot['is_batch']:
        self._retry_batch_task(snapshot)
    else:
        self._retry_single_task(snapshot)

def _retry_single_task(self, snapshot):
    if not os.path.exists(snapshot['input_path']):
        QMessageBox.warning(self, '文件不存在', f'源文件已不存在：{snapshot["input_path"]}')
        return
    new_task_id = uuid4()[:8]
    snapshot['task_id'] = new_task_id
    self.task_list_widget.reset_task(old_task_id, new_task_id, snapshot)
    self.task_manager.submit_task(new_task_id, ...)
```

**Tab 切换逻辑改动**：
```python
def _on_conversion_type_changed(self, _=None):

    self.file_list_widget.clear_files()
    # 更新拖拽区提示文字
```

```python
def _cancel_all(self):
```

## 3. 技术选型与依赖

| 技术点 | 选型 | 说明 |
|--------|------|------|
| 分栏布局 | `QSplitter` | 支持用户拖动调整比例，比固定 `QHBoxLayout` 更灵活 |
| 任务项状态颜色 | QSS `objectName` 动态切换 | 与现有主题系统保持一致，避免引入 QPalette 硬编码 |
| 输出文件点击打开 | `os.startfile(path)` | 沿用现有实现，无需新依赖 |
| Tab 切换回滚 | 记录 `_last_tab_index` | 用户取消切换时恢复，需阻止信号递归 |
| 无需新增第三方库 | — | 本次改动完全在 PySide6 范围内 |

## 4. 开发顺序

> 按依赖关系从底层到顶层逐步推进，每步均可独立验证。

### Step 1 — 简化 `FileListItem`（drop_zone.py）

- 移除进度条、状态标签及相关方法
- 调整布局为简洁单行
- 验证：拖拽文件后列表显示正常，删除按钮正常工作

### Step 2 — 简化 `FileListWidget`

- 剥离 `task_id`/`done` 字段、所有任务相关方法
- 保持公共接口（`add_files`、`remove_file` 等）不变
- 验证：单元逻辑正确，`file_count_changed` 信号正常

### Step 3 — 实现 `TaskListItem`

- 布局、状态枚举、视觉映射
- `set_state()` 方法
- 重试/重新开始按钮的显示逻辑
- `retry_requested` 信号
- 验证：手动实例化、调用 `set_state()` 测试各状态外观

### Step 4 — 实现 `TaskListWidget`

- 容器布局、顶部操作栏（标题 + 清除已完成按钮）
- `add_task()`、`on_task_started/progress/finished()`、`cancel_all_pending()`、`clear_completed()`
- `_items`、`_snapshots` 内部状态管理
- `retry_requested` 信号冒泡
- 验证：模拟添加任务、更新状态、点击清除

### Step 5 — 重构 `main_window.py` 布局

- 引入 `QSplitter`，划分左右两侧 widget
- 左侧：拖拽区 + 文件列表；右侧：任务列表
- 操作按钮行、Tab 保持全宽
- 验证：UI 正常显示，分栏可拖动

### Step 6 — 重构 `main_window.py` 数据流

- `_start_conversion()` 改为提交后移除文件、添加任务条目
- 三个任务回调改为驱动 `TaskListWidget`
- 接入 `retry_requested` 信号，实现 `_on_retry_requested()`
- 改造 `_cancel_all()` 驱动 `cancel_all_pending()`
- 验证：完整转换流程端到端测试

### Step 7 — Tab 切换确认弹窗

- 记录 `_last_tab_index`，切换时判断文件列表是否非空
- 弹确认框，取消则回滚 Tab（用 `blockSignals` 防止递归）
- 验证：有文件时切换弹框出现，无文件时直接切换

### Step 8 — 端到端测试 + 样式微调

- 按验收标准 AC-1 ~ AC-8 逐条验证
- 检查 QSS 中是否需要为 `TaskListItem` 新增样式规则
- 检查暗色/亮色主题下颜色标记是否正确
