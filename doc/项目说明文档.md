## 1. 项目基本信息

### 项目名称

**DocFlow**

### 当前版本

**V0.2.1**

### 项目目标 / 功能概述

DocFlow 是一款面向 Windows 用户的**本地离线文档格式转换工具**。用户无需联网，文件全程不离开本地，即可在 Word、PPT、PDF、图片等常用格式之间进行快速批量转换。

软件提供图形化界面（GUI），支持拖拽文件或目录，操作简单直观，转换任务在后台多线程异步执行，不阻塞界面响应。

**核心特性：**

- 支持 7 种主流文档格式互转
- 本地运行，无需联网，保护文件隐私
- 支持拖拽目录，自动识别符合条件的文件
- 多文件并发转换，实时显示每文件进度
- 支持自定义输出目录
- 深色 / 浅色主题切换，持久化保存偏好
- 支持打包为单文件 EXE，开箱即用

### 背景和动机

日常工作中常需要在 Word、PPT、PDF、图片等格式间互转，现有在线工具存在文件隐私泄露风险，而专业软件成本高、操作复杂。DocFlow 旨在提供一款**轻量、免费、纯本地**的替代方案，降低非技术用户的使用门槛。

### 2.3 各模块职责说明

#### `main.py` — 应用入口

初始化日志、创建 `QApplication` 和 `MainWindow`，启动事件循环。同时处理 PyInstaller 打包后的资源路径适配（`sys.frozen` 判断）。

#### `core/conversion_registry.py` — 转换类型注册表

全局单一数据源，统一存储 7 种转换类型的元数据（`id`、`label`、`input_exts`、`output_ext`、`hint_text`、`tab`、`is_batch`），提供 `get_by_id()` / `get_by_tab()` / `get_all_input_exts()` 等查询 API，消除多处重复定义。

#### `core/task_manager.py` — 任务调度器

使用 `QThreadPool`（最多 4 线程）管理转换任务，封装 `ConvertTask`（单文件任务）和 `BatchImageTask`（批量合并任务）两类 `QRunnable`，通过 Qt Signal 向 UI 上报进度和结果。维护 `_cancelled_ids` 集合，确保已取消任务的所有后续信号被静默丢弃，不影响 UI 状态。

#### `core/converter/dispatcher.py` — 路由调度器

`dispatch()` 函数按 `conversion_type` 字符串将任务分派到对应转换函数；`dispatch_batch()` 处理批量合并类转换。

#### `core/converter/office_converter.py` — Office COM 层

通过 `win32com.DispatchEx` 调用本机 Word / PPT（优先 Microsoft Office，其次 WPS），使用全局互斥锁 `_office_lock` 和 `CoInitialize / CoUninitialize` 保证多线程下 COM 操作的安全性。

#### `core/converter/pdf_converter.py` — PDF 转换器

- `pdf_to_word`：基于 `pdf2docx` 库
- `pdf_to_ppt`：使用 `PyMuPDF` 将每页渲染为图片后插入 `python-pptx` 幻灯片
- `pdf_to_images`：使用 `PyMuPDF` 渲染，多页自动在输出目录新建子目录存放

#### `core/converter/image_converter.py` — 图片转换器

- `images_to_pdf`：使用 `Pillow` 将多张图片合并为单个 PDF
- `images_to_word`：使用 `python-docx` 将多张图片逐一插入 Word 文档

#### `ui/main_window.py` — 主窗口

三 Tab 布局（📄 文档转换 / 📕 PDF 转换 / 🖼️ 图片转换），采用 `QSplitter` 左右分栏显示文件列表和任务历史列表。提供文件拖拽区、下拉转换类型选择、输出目录选择、"开始转换"/"取消全部"按钮，调用 `TaskManager` 下发任务，并订阅进度 Signal 驱动右侧任务列表更新状态。切换转换类型时，若新旧类型接受的扩展名不同且文件列表非空，弹出确认框后清空不匹配文件。

#### `ui/theme_manager.py` — 主题管理器

加载 `styles.qss`（深色）和 `styles_light.qss`（浅色），使用 `QSettings` 持久化用户的主题偏好，提供 `toggle()` / `apply()` / `get_button_icon()` 接口。

#### `ui/widgets/drop_zone.py` — 拖拽区组件

`DropZone`：处理 `dragEnterEvent` / `dropEvent`，支持拖入文件和目录。
`FileListItem`：单文件行组件，使用 SVG 矢量图标显示文件类型，含文件名、大小和移除按钮，兼容 PyInstaller 打包路径。

#### `ui/widgets/file_list_widget.py` — 文件列表组件

`FileListWidget` 可滚动容器，管理待转换文件的增删查。文件提交转换后从列表移除，仅展示等待处理的文件。

#### `ui/widgets/task_list_widget.py` — 任务历史列表组件

`TaskListItem`：单任务行，展示状态圆点、转换标题、进度条、状态文字及操作按钮（失败→"重试"，取消→"重新开始"）；完成后输出文件名可点击打开。
`TaskListWidget`：可滚动任务历史容器，提供"清除已完成"按钮，向主窗口发出 `retry_requested` 信号。

#### `ui/widgets/help_dialog.py` — 帮助弹窗

内置 HTML 使用说明，通过 `QTextBrowser` 渲染展示，用户点击主窗口右上角 `?` 按钮触发。

#### `utils/file_utils.py` — 文件工具

- 文件类型识别（委托 `conversion_registry` 判断）
- `get_output_path()`：自动加数字后缀，防止覆盖已有文件
- `get_friendly_size()`：将字节数转为人类可读的文件大小字符串

#### `utils/logger.py` — 日志工具

初始化 `logging` 模块，写入 `log.txt`，提供全局 `logger` 单例和 `log_conversion()` 快捷函数，记录每次转换操作的结果。

## 3. 技术架构

### 3.1 总体架构说明

DocFlow 采用**分层架构**，将 UI 层、业务核心层和工具层明确分离：

```
┌─────────────────────────────────────────┐
│              UI 层（ui/）                │
│  MainWindow / ThemeManager / Widgets    │
└────────────────────┬────────────────────┘
                     │ 调用
┌────────────────────▼────────────────────┐
│          业务核心层（core/）              │
│  ConversionRegistry / TaskManager       │
│  Dispatcher → Converters               │
└────────────────────┬────────────────────┘
                     │ 使用
┌────────────────────▼────────────────────┐
│           工具层（utils/）               │
│       FileUtils / Logger               │
└─────────────────────────────────────────┘
```

**数据流：**

1. 用户在 UI 拖入文件 → `MainWindow` 校验类型 → 加入 `FileListWidget`
2. 点击"开始转换" → `TaskManager` 创建任务对象加入 `QThreadPool`
3. 工作线程调用 `dispatcher.dispatch()` → 路由到具体 Converter 函数
4. Converter 执行完成后，通过 Qt Signal 通知 UI 更新进度/状态

### 3.2 技术栈

| 类别       | 选型                              |
| ---------- | --------------------------------- |
| 开发语言   | Python 3.11+                      |
| GUI 框架   | PySide6（Qt 6）                   |
| Office 调用| pywin32（Win32 COM）              |
| PDF 渲染   | PyMuPDF（fitz）                   |
| PDF→Word   | pdf2docx                          |
| 图片处理   | Pillow                            |
| Word 生成  | python-docx                       |
| PPT 生成   | python-pptx                       |
| 打包工具   | PyInstaller                       |
| 运行平台   | Windows 10 / 11（64位）           |

完整依赖见 [requirements.txt](../requirements.txt)。

### 3.3 主要设计模式和工程策略

| 模式 / 策略         | 说明                                                                                                              |
| ------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **注册表模式**      | `ConversionRegistry` 作为转换类型的单一数据源，消除 UI / 业务层中重复的格式定义，统一查询 API                    |
| **命令 + 调度模式** | `Dispatcher` 将 `conversion_type` 字符串映射到对应函数，UI 和任务管理器无需感知具体转换实现                      |
| **异步队列任务**    | 使用 `QThreadPool` + `QRunnable` 实现多线程并发，通过 Qt Signal/Slot 机制跨线程安全通信                           |
| **COM 线程安全**    | Office COM 调用使用 `threading.Lock` 串行化 + `CoInitialize/CoUninitialize` 在 `finally` 中确保执行，避免多线程崩溃 |
| **路径自适应**      | 通过 `sys.frozen` 判断运行环境，分别处理开发模式和 PyInstaller 打包后的资源路径                                   |

## 5. 使用说明

### 5.1 系统要求

- **操作系统**：Windows 10 / 11（64 位）
- **可选依赖**：若需使用 Word / PPT → PDF 转换，须已安装 **Microsoft Office** 或 **WPS**

### 5.2 安装 / 部署方法

#### 方式一：直接运行 EXE（推荐普通用户）

从 Release 页面下载 `DocFlow.exe`，放置至任意目录，双击运行即可。  
首次启动因解压内置依赖，需等待数秒，属正常现象。

#### 方式二：源码运行（开发者）

```bash
# 1. 克隆仓库
git clone <仓库地址>
cd DocFlow

# 2. 创建虚拟环境（推荐 Python 3.11+）
python -m venv .venv
.venv\Scripts\activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 运行
python main.py
```

### 5.3 打包为 EXE

```bash
# 使用已配置好的打包脚本（onefile 模式）
scripts\pack.bat

# 或手动执行 PyInstaller
pyinstaller main.spec
```

打包产物位于 `dist/` 目录。详见 [doc/DocFlow打包说明.md](DocFlow打包说明.md)。

### 5.4 运行方法

1. 启动软件后，顶部选择功能选项卡（文档 / PDF / 图片转换）
2. 在下拉菜单中选择具体的转换方向
3. 拖拽文件或目录至中央虚线框，或单击虚线框通过文件浏览器选择
4. （可选）点击左下角"📁 选择输出目录"指定保存位置；默认保存至源文件所在目录
5. 点击"▶ 开始转换"，等待任务完成
6. 转换完成后状态显示"✓ 完成"，若勾选自动打开则自动启动输出文件

### 5.5 配置说明

| 配置项         | 位置                                               | 说明                               |
| -------------- | -------------------------------------------------- | ---------------------------------- |
| 主题偏好       | 系统注册表（QSettings，自动保存）                  | 深色 / 浅色，重启后保持            |
| 界面样式       | `resources/styles.qss` / `resources/styles_light.qss` | 可手动编辑 QSS 自定义样式      |
| 运行日志       | `log.txt`（与 EXE 同目录）                         | 记录每次转换的详细错误信息         |
| 最大并发线程数 | `core/task_manager.py` 中 `QThreadPool.setMaxThreadCount(4)` | 默认 4，可按需调整    |

## 6. 开发和维护指南

### 6.1 分支管理

| 分支          | 用途                                     |
| ------------- | ---------------------------------------- |
| `main`        | 稳定版本，对应最新 Release               |
| `V<version>`  | 各版本开发分支，如 `V0.1.1`              |
| `feature/xxx` | 新功能开发，从当前开发分支切出，完成后 PR|
| `bugfix/xxx`  | Bug 修复分支                             |

> 请勿直接向 `main` 分支提交代码，所有改动须通过 Pull Request 合并。

### 6.2 提交规范

提交信息遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
<type>: <简短描述>

type 可选值：
  feat     新功能
  fix      Bug 修复
  chore    构建/依赖/工具相关
  refactor 代码重构（不影响功能）
  docs     文档更新
  style    样式/格式调整
  test     测试相关
```

示例：
```
feat: 支持 Markdown 转 PDF
fix: 修复 PDF 多页转图片路径错误
chore: 更新 PyMuPDF 至 1.25.0
```

### 6.3 文档更新规范

- **新增功能或重要变更**：同步更新 `doc/changelog.md`，按"版本号 > 类型 > 描述"三级结构填写
- **面向用户的使用变化**：同步更新 `doc/readme.md`
- **架构 / 设计决策**：在 `doc/<版本号>/` 目录下存放分析报告和设计文档

### 6.4 新增转换类型

1. 在 `core/conversion_registry.py` 中按现有格式添加新的转换条目
2. 在对应的 Converter 文件（或新建 `*_converter.py`）中实现转换函数
3. 在 `core/converter/dispatcher.py` 的 `dispatch()` / `dispatch_batch()` 中注册路由
4. 在 `ui/main_window.py` 中为对应 Tab 的下拉菜单添加选项
5. 更新 `requirements.txt`（如有新依赖）
6. 更新 `doc/readme.md` 中的格式支持表格

### 6.5 重构注意事项

- **跨层调用**：严格遵守分层原则，UI 层只调用 `core/` 接口，不直接引用 Converter
- **COM 线程安全**：所有 Office COM 操作必须在 `_office_lock` 互斥锁保护下执行，且在 `finally` 中调用 `CoUninitialize`
- **PyInstaller 兼容**：新增资源文件须在 `main.spec` 的 `datas` 中注册；路径读取须使用适配 `sys.frozen` 的辅助函数
- **注册表优先**：涉及转换类型的元数据修改，一律在 `conversion_registry.py` 中进行，不要在 UI 层硬编码格式列表

| 中     | 图片格式转换               | 支持 PNG ↔ JPG ↔ WebP 等图片格式互转             |
| 低     | 自动更新检测               | 启动时检查是否有新版本发布                        |
| 低     | 国际化（i18n）             | 支持英文等多语言界面                              |

### 7.2 预期改进方向

- **跨平台支持**：当前 Windows 专用（COM 依赖），计划调研在 macOS / Linux 上的可行替代方案（如 LibreOffice 命令行）
- **Office 独立转换**：移除对本机 Office/WPS 的强依赖，探索使用纯 Python 库（如 `docx2pdf` 配合 LibreOffice）实现无 Office 环境下的文档转换
- **性能优化**：大文件 / 大批量转换时的内存管理优化，考虑进程级隔离替代线程级隔离以规避 COM 限制
- **单元测试覆盖**：为 `core/` 层的转换逻辑补充自动化测试，提高重构安全性
- **UI 体验提升**：文件列表拖拽排序、批量移除、转换完成声音提示等细节优化
