## Step 1｜新建转换类型注册表

**完成日期**：2026-03-30
**操作类型**：纯新增文件，零风险

### 目标
建立 `CONVERSION_REGISTRY` 作为全局唯一的转换类型知识库，
替代原先散落在三处的重复数据（`SUPPORTED_EXTENSIONS`、`CONVERSION_MAP`、`_HINT_MAP`、`ext_map`）。

### 新增文件

#### `core/conversion_registry.py`

**数据结构**：`CONVERSION_REGISTRY` 为有序列表，共 7 个条目，每个条目包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | str | 转换类型唯一标识符，与 task_manager 中 `conversion_type` 字符串对齐 |
| `label` | str | 下拉框显示文字 |
| `input_exts` | list | 接受的输入文件扩展名（小写，含点号） |
| `output_ext` | str | 输出文件扩展名 |
| `hint_text` | str | 拖拽区底部提示文字 |
| `tab` | str | 所属 Tab 分组：`'doc'` / `'pdf'` / `'image'` |
| `file_type` | str | 文件分类标识，供 `get_file_type()` 使用 |
| `is_batch` | bool | 是否为批量合并模式 |

**注册的 7 种转换类型**（按 Tab 分组）：

| Tab | ID | 标签 |
|-----|-----|------|
| doc | `word_to_pdf` | Word → PDF |
| doc | `ppt_to_pdf` | PPT → PDF |
| pdf | `pdf_to_word` | PDF → Word |
| pdf | `pdf_to_ppt` | PDF → PPT |
| pdf | `pdf_to_image` | PDF → 图片 |
| image | `images_to_pdf` | 图片 → PDF（合并） |
| image | `images_to_word` | 图片 → Word |

**对外提供的函数**：
- `get_by_id(id)` → 按 ID 查询单个条目
- `get_by_tab(tab_name)` → 按 Tab 查询条目列表
- `get_all_input_exts()` → 返回所有输入扩展名集合（供 Step 3 的 `is_supported_file()` 使用）
- `get_file_type_from_ext(ext)` → 根据扩展名返回文件分类（供 Step 3 的 `get_file_type()` 使用）

### 修改的文件

无。本步骤不触动任何现有文件。

### 验证结果

```
条目数: 7
word_to_pdf: {'id': 'word_to_pdf', 'label': 'Word → PDF', 'input_exts': ['.doc', '.docx'], ...}
pdf tab: ['pdf_to_word', 'pdf_to_ppt', 'pdf_to_image']
所有扩展名: ['.doc', '.docx', '.jpeg', '.jpg', '.pdf', '.png', '.ppt', '.pptx']
```

### 已知不一致项（供后续步骤参考）

`file_utils.py` 中 `CONVERSION_MAP` 的 key 使用 `image_to_pdf`（单数），
而 `task_manager.py` 与 `main_window.py` 实际使用的是 `images_to_pdf`（复数）。
注册表统一采用**复数形式**（`images_to_pdf`、`images_to_word`），与运行路径保持一致。

`file_utils.py` 中的 `CONVERSION_MAP` 保留原样，待 Step 3 统一清理。

## Step 4｜新建转换路由调度器

**完成日期**：2026-03-30
**操作类型**：纯新增文件，零风险

### 目标

新建 `core/converter/dispatcher.py`，将 `task_manager.py` 中的两处路由逻辑（单文件转换 `_do_convert()` + 批量图片转换 `BatchImageTask.run()`）提取为独立的纯函数。此步仅新增文件，不修改任何现有代码。

### 新建文件

#### `core/converter/dispatcher.py`

**函数 1：`dispatch(conversion_type, input_path, output_dir, options) -> str`**

将 `ConvertTask._do_convert()` 及其私有辅助 `_convert_to_pdf()` 的全部 `if-elif` 路由逻辑原样提取为独立顶层函数。支持的转换类型：

| conversion_type | 调用的 converter 函数 | 输出格式 |
|----------------|----------------------|--------|
| `to_pdf` | `_convert_to_pdf()`（兼容旧版，按扩展名路由） | `.pdf` |
| `word_to_pdf` | `word_to_pdf()` | `.pdf` |
| `ppt_to_pdf` | `ppt_to_pdf()` | `.pdf` |
| `pdf_to_word` | `pdf_to_word()` | `.docx` |
| `pdf_to_ppt` | `pdf_to_ppt()` | `.pptx` |
| `pdf_to_image` | `pdf_to_images()`（多页返回目录，单页返回文件） | `.png/.jpg` |
| `image_to_pdf` | `images_to_pdf([input_path], ...)` | `.pdf` |
| `image_to_word` | `images_to_word([input_path], ...)` | `.docx` |

**函数 2：`dispatch_batch(conversion_type, image_paths, output_path) -> str`**

将 `BatchImageTask.run()` 中的批量图片合并路由提取为独立函数，支持：
- `images_to_pdf` → 调用 `images_to_pdf()`
- `images_to_word` → 调用 `images_to_word()`

**私有辅助函数：`_convert_to_pdf(input_path, ext, output_dir) -> str`**

保持 `to_pdf` 兼容行为：`.doc/.docx` 走 `word_to_pdf`，`.ppt/.pptx` 走 `ppt_to_pdf`，其余抛出 `RuntimeError`。

### 与 task_manager.py 的差异

| 方面 | task_manager.py 原代码 | dispatcher.py |
|-----|----------------------|--------------|
| 代码位置 | `ConvertTask._do_convert()` 实例方法 | 顶层纯函数 |
| 依赖 `self` | 是（通过 `self.input_path` 等） | 否（所有参数显式传入） |
| `to_pdf` 兼容分支 | 有（调用 `self._convert_to_pdf(ext)`） | 保留（调用私有 `_convert_to_pdf()`） |
| `task_manager.py` 原代码 | **保持不变** | 新增，不删除原逻辑 |

### 改后完整文件（`core/task_manager.py`）

```python
"""任务管理器 - 管理转换任务的线程池调度"""
import os
import traceback
from PySide6.QtCore import QObject, Signal, QRunnable, QThreadPool, Slot

from utils.logger import logger
from core.converter.dispatcher import dispatch, dispatch_batch


class TaskSignals(QObject):
    """任务信号"""
    started = Signal(str)
    progress = Signal(str, int)
    finished = Signal(str, bool, str, str)
    all_done = Signal()


class ConvertTask(QRunnable):
    """单个转换任务"""

    def __init__(self, task_id, input_path, conversion_type, output_dir=None, options=None):
        super().__init__()
        self.task_id = task_id
        self.input_path = input_path
        self.conversion_type = conversion_type
        self.output_dir = output_dir or os.path.dirname(input_path)
        self.options = options or {}
        self.signals = TaskSignals()
        self._cancelled = False

    def cancel(self):
        self._cancelled = True

    @Slot()
    def run(self):
        if self._cancelled:
            self.signals.finished.emit(self.task_id, False, '已取消', '')
            return
        self.signals.started.emit(self.task_id)
        self.signals.progress.emit(self.task_id, 10)
        try:
            result = self._do_convert()
            self.signals.progress.emit(self.task_id, 100)
            self.signals.finished.emit(self.task_id, True, f'转换成功: {os.path.basename(result)}', result)
        except Exception as e:
            logger.error(f'任务 {self.task_id} 失败: {traceback.format_exc()}')
            self.signals.finished.emit(self.task_id, False, str(e), '')

    def _do_convert(self) -> str:
        return dispatch(self.conversion_type, self.input_path, self.output_dir, self.options)


class BatchImageTask(QRunnable):
    """批量图片合并任务（多图合一）"""

    def __init__(self, task_id, image_paths, conversion_type, output_path):
        super().__init__()
        self.task_id = task_id
        self.image_paths = image_paths
        self.conversion_type = conversion_type
        self.output_path = output_path
        self.signals = TaskSignals()

    @Slot()
    def run(self):
        self.signals.started.emit(self.task_id)
        self.signals.progress.emit(self.task_id, 10)
        try:
            dispatch_batch(self.conversion_type, self.image_paths, self.output_path)
            self.signals.progress.emit(self.task_id, 100)
            self.signals.finished.emit(
                self.task_id, True,
                f'转换成功: {os.path.basename(self.output_path)}',
                self.output_path
            )
        except Exception as e:
            logger.error(f'批量任务 {self.task_id} 失败: {traceback.format_exc()}')
            self.signals.finished.emit(self.task_id, False, str(e), '')


class TaskManager(QObject):
    """任务管理器 - 管理所有转换任务"""
    task_started = Signal(str)
    task_progress = Signal(str, int)
    task_finished = Signal(str, bool, str, str)
    all_tasks_done = Signal()

    def __init__(self, parent=None):
        super().__init__(parent)
        self.thread_pool = QThreadPool.globalInstance()
        self.thread_pool.setMaxThreadCount(4)
        self._tasks = {}
        self._pending_count = 0

    def submit_task(self, task_id, input_path, conversion_type, output_dir=None, options=None):
        task = ConvertTask(task_id, input_path, conversion_type, output_dir, options)
        task.signals.started.connect(self._on_task_started)
        task.signals.progress.connect(self._on_task_progress)
        task.signals.finished.connect(self._on_task_finished)
        self._tasks[task_id] = task
        self._pending_count += 1
        self.thread_pool.start(task)

    def submit_batch_image_task(self, task_id, image_paths, conversion_type, output_path):
        task = BatchImageTask(task_id, image_paths, conversion_type, output_path)
        task.signals.started.connect(self._on_task_started)
        task.signals.progress.connect(self._on_task_progress)
        task.signals.finished.connect(self._on_task_finished)
        self._tasks[task_id] = task
        self._pending_count += 1
        self.thread_pool.start(task)

    def cancel_all(self):
        for task in self._tasks.values():
            if hasattr(task, 'cancel'):
                task.cancel()
        self._tasks.clear()
        self._pending_count = 0

    def _on_task_started(self, task_id):
        self.task_started.emit(task_id)

    def _on_task_progress(self, task_id, percent):
        self.task_progress.emit(task_id, percent)

    def _on_task_finished(self, task_id, success, message, output_path):
        self.task_finished.emit(task_id, success, message, output_path)
        self._pending_count -= 1
        if self._pending_count <= 0:
            self._pending_count = 0
            self.all_tasks_done.emit()
```

### 新建文件

#### `ui/widgets/file_list_widget.py`

**类：`FileListWidget(QWidget)`**

**信号**

| 信号 | 类型 | 说明 |
|------|------|------|
| `file_count_changed` | `Signal(int)` | 文件数量变化时发出 |
| `file_removed` | `Signal(str)` | 单个文件被移除时发出（携带路径） |

**属性**

| 属性 | 返回值 | 来源 |
|------|--------|------|
| `file_paths` | `list[str]` | 所有文件路径（按添加顺序） |
| `pending_paths` | `list[str]` | 尚未完成转换的文件路径 |
| `task_id_map` | `dict` | `{task_id: [file_path, ...]}` 映射 |

**公开方法**

| 方法 | 签名 | 迁移来源 |
|------|------|---------|
| `add_files` | `(file_paths, accepted_exts) -> list[str]` | `MainWindow._add_files()`（返回被拒文件名） |
| `remove_file` | `(file_path) -> None` | `MainWindow._remove_file()` |
| `clear_files` | `() -> None` | `MainWindow._clear_files()` |
| `collect_files_from_dir` | `(dir_path, accepted_exts) -> list[str]` | `MainWindow._collect_files_from_dir()` |

| `reset_all_tasks` | `() -> None` | 合并自 `MainWindow._cancel_all()` 中的 widget 重置部分 |
| `update_task_state` | `(task_id, state, percent, success, output_path) -> list[str]` | 合并自三个 `_on_task_*` 回调的 widget 操作 |

**`update_task_state` 参数说明**

| 参数 | 类型 | state='started' | state='progress' | state='finished' |
|------|------|----------------|-----------------|----------------|
| `state` | str | `'started'` | `'progress'` | `'finished'` |
| `percent` | int | 忽略 | 进度值 0-100 | 忽略 |
| `success` | bool | 忽略 | 忽略 | 是否成功 |
| `output_path` | str | 忽略 | 忽略 | 输出路径 |
| **返回值** | list | `[]` | `[]` | 成功时的输出路径列表 |

**设计说明**

- `add_files()` 不再弹窗，只返回被拒文件名列表，由调用方决定是否弹窗
- `remove_file()` 自动 emit `file_removed(path)` 信号
- `update_task_state()` 合并了原来三个 `@Slot` 回调中 widget 更新部分，不再分离
- 批量任务场景下多个文件共享同一 `task_id`，`update_task_state` 不 break，遍历全部匹配项
- 收集 `output_path` 并以列表返回，供 main_window 做「转换完成后自动打开」
