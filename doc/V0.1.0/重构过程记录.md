## Step 1｜新建转换类型注册表

**完成日期**：2026-03-30
**操作类型**：纯新增文件，零风险

### 目标
建立 `CONVERSION_REGISTRY` 作为全局唯一的转换类型知识库，
替代原先散落在三处的重复数据（`SUPPORTED_EXTENSIONS`、`CONVERSION_MAP`、`_HINT_MAP`、`ext_map`）。

### 新增文件

#### `core/conversion_registry.py`

**数据结构**：`CONVERSION_REGISTRY` 为有序列表，共 7 个条目，每个条目包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | str | 转换类型唯一标识符，与 task_manager 中 `conversion_type` 字符串对齐 |
| `label` | str | 下拉框显示文字 |
| `input_exts` | list | 接受的输入文件扩展名（小写，含点号） |
| `output_ext` | str | 输出文件扩展名 |
| `hint_text` | str | 拖拽区底部提示文字 |
| `tab` | str | 所属 Tab 分组：`'doc'` / `'pdf'` / `'image'` |
| `file_type` | str | 文件分类标识，供 `get_file_type()` 使用 |
| `is_batch` | bool | 是否为批量合并模式 |

**注册的 7 种转换类型**（按 Tab 分组）：

| Tab | ID | 标签 |
|-----|-----|------|
| doc | `word_to_pdf` | Word → PDF |
| doc | `ppt_to_pdf` | PPT → PDF |
| pdf | `pdf_to_word` | PDF → Word |
| pdf | `pdf_to_ppt` | PDF → PPT |
| pdf | `pdf_to_image` | PDF → 图片 |
| image | `images_to_pdf` | 图片 → PDF（合并） |
| image | `images_to_word` | 图片 → Word |

**对外提供的函数**：
- `get_by_id(id)` → 按 ID 查询单个条目
- `get_by_tab(tab_name)` → 按 Tab 查询条目列表
- `get_all_input_exts()` → 返回所有输入扩展名集合（供 Step 3 的 `is_supported_file()` 使用）
- `get_file_type_from_ext(ext)` → 根据扩展名返回文件分类（供 Step 3 的 `get_file_type()` 使用）

### 修改的文件

无。本步骤不触动任何现有文件。

### 验证结果

```
条目数: 7
word_to_pdf: {'id': 'word_to_pdf', 'label': 'Word → PDF', 'input_exts': ['.doc', '.docx'], ...}
pdf tab: ['pdf_to_word', 'pdf_to_ppt', 'pdf_to_image']
所有扩展名: ['.doc', '.docx', '.jpeg', '.jpg', '.pdf', '.png', '.ppt', '.pptx']
```

### 已知不一致项（供后续步骤参考）

`file_utils.py` 中 `CONVERSION_MAP` 的 key 使用 `image_to_pdf`（单数），
而 `task_manager.py` 与 `main_window.py` 实际使用的是 `images_to_pdf`（复数）。
注册表统一采用**复数形式**（`images_to_pdf`、`images_to_word`），与运行路径保持一致。

`file_utils.py` 中的 `CONVERSION_MAP` 保留原样，待 Step 3 统一清理。

## Step 2｜main_window 切换读取注册表

**完成日期**：2026-03-30
**操作类型**：修改现有文件，低风险

### 目标
删除 `main_window.py` 中三处硬编码的转换类型数据，统一改为从 `conversion_registry` 读取。

### 修改文件

#### `ui/main_window.py`

**改动 1：新增 import**

```python
# 新增
from core.conversion_registry import get_by_id, get_by_tab
```

**改动 2：`_create_tab_page()` 中 combo 条目改为注册表驱动**

删除原来三个 `if tab_type == ...` 分支（共 7 行 `combo.addItem()` 硬编码），替换为：

```python
# 改后
for entry in get_by_tab(tab_type):
    combo.addItem(entry['label'], entry['id'])
```

**改动 3：删除 `_HINT_MAP` 类变量，`_on_conversion_type_changed()` 改为查注册表**

删除 10 行的 `_HINT_MAP` 字典，修改方法体：

```python
# 改后
def _on_conversion_type_changed(self, _=None):
    conversion_type = self._get_current_conversion_type()
    entry = get_by_id(conversion_type)
    hint = entry['hint_text'] if entry else '支持 Word、PPT、PDF、图片文件'
    self.drop_zone.set_hint(hint)
    self._filter_files_for_current_type()
```

**改动 4：`_get_accepted_extensions()` 删除 `ext_map`，改为查注册表**

删除 10 行的 `ext_map` 字典，替换为：

```python
# 改后
def _get_accepted_extensions(self, conversion_type: str) -> list:
    entry = get_by_id(conversion_type)
    return entry['input_exts'] if entry else []
```

> **注意**：原 `ext_map` 中有 `'to_pdf'` 兼容键（`['.doc', '.docx', '.ppt', '.pptx']`），注册表中无此条目，`get_by_id('to_pdf')` 返回 `None`，`_get_accepted_extensions` 返回 `[]`（即不过滤）。
> 经确认 `_get_current_conversion_type()` 的默认回退值 `'to_pdf'` 仅在 `combo` 找不到时触发（正常使用不可达），行为与原来一致，安全。
