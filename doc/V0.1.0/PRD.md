# 📄 文档转换工具 PRD（Windows 桌面应用）

## 一、产品概述

### 1.1 产品名称

DocFlow（暂定）

### 1.2 产品定位

一款面向普通用户的 **本地离线文档转换工具**，支持 Office / WPS 环境，提供高保真格式转换能力，强调：

* 操作简单（傻瓜式）
* 界面美观（接近商业软件）
* 批量处理能力
* 拖拽式操作

---

## 二、目标用户

* 普通办公用户（非技术）
* 学生 / 教师
* 行政 / 文员
* 有文档转换需求但不愿使用在线工具的用户

---

## 三、核心需求（功能范围）

---

## 3.1 功能总览

| 功能模块           | 描述         |
| -------------- | ---------- |
| Word/PPT → PDF | 高保真转换      |
| PDF → Word/PPT | 尽可能还原结构    |
| 图片 → PDF/Word  | 支持多图合并     |
| PDF → 图片       | 多页拆分       |
| 批量处理           | 多文件同时转换    |
| 拖拽上传           | 文件拖入界面即可处理 |

---

## 3.2 功能详细说明

---

### 3.2.1 Word / PPT → PDF

#### 输入：

* `.doc`, `.docx`, `.ppt`, `.pptx`

#### 输出：

* `.pdf`

#### 要求：

* 使用本地 Office 或 WPS
* 保证高保真（排版一致）
* 支持批量

---

### 3.2.2 PDF → Word / PPT

#### 输入：

* `.pdf`

#### 输出：

* `.docx` / `.pptx`

#### 要求：

* 优先使用解析库（如 pdf2docx）
* 若失败需提示用户：

  > “该PDF可能为扫描件，转换效果有限”

---

### 3.2.3 图片 → PDF / Word

#### 输入：

* `.jpg`, `.png`, `.jpeg`

#### 功能：

* 多图片按顺序合并
* 支持拖拽排序

#### 输出：

* PDF（每张图片一页）
* Word（插入图片）

---

### 3.2.4 PDF → 图片

#### 输入：

* `.pdf`

#### 输出：

* `.jpg` 或 `.png`

#### 要求：

* 每页生成一张图片
* 支持设置：

  * DPI（默认 200）
  * 输出格式

---

### 3.2.5 批量处理

#### 要求：

* 支持多文件同时转换
* 每个文件独立进度
* 可全部开始 / 全部取消

---

### 3.2.6 拖拽上传

#### 要求：

* 支持文件拖入窗口
* 自动识别文件类型
* 自动匹配转换类型

---

## 四、非功能需求

---

## 4.1 用户体验（UX）

### 必须满足：

* 操作路径 ≤ 3步
* 所有操作有明确提示
* 所有错误可读（不能报Python异常）

---

### 提示示例：

| 场景        | 提示                       |
| --------- | ------------------------ |
| 未安装Office | “未检测到 Office 或 WPS，请先安装” |
| 转换失败      | “转换失败，请检查文件是否损坏”         |
| 扫描PDF     | “检测到扫描PDF，转换效果可能较差”      |

---

## 4.2 UI设计要求

### 风格：

* 现代化（类似轻量版 Adobe 工具）
* 简洁（主界面不超过3个核心区域）
* 美观（使用图标和颜色区分状态）

---

### 界面结构：

```
---------------------------------
| 标题栏                        |
---------------------------------
| 功能选择 Tabs                 |
| [PDF转换] [图片转换] [文档转换]|
---------------------------------
| 拖拽区域（大面积）            |
| “拖拽文件到这里”              |
---------------------------------
| 文件列表 + 状态               |
---------------------------------
| 操作按钮（开始/停止）         |
---------------------------------
```

---

### 必须支持：

* 拖拽
* 进度条
* 状态提示（成功/失败）
* 图标提示（文件类型）

---

## 4.3 性能要求

* 单文件转换 ≤ 5秒（普通文档）
* 批量处理不卡UI（必须多线程）
* UI不卡顿

---

## 4.4 兼容性

| 项目      | 要求                  |
| ------- | ------------------- |
| 操作系统    | Windows 7 / 10 / 11 |
| Office  | 支持                  |
| WPS     | 支持                  |
| 无Office | ❌ 不支持               |

---

## 五、技术方案（AI实现约束）

---

## 5.1 技术栈

### UI：

* PySide6（Qt）

---

### 核心转换：

| 功能       | 技术          |
| -------- | ----------- |
| Office转换 | win32com    |
| PDF解析    | pdf2docx    |
| PDF转图    | pdf2image   |
| 图片处理     | Pillow      |
| Word生成   | python-docx |

---

## 5.2 转换策略

```
优先级：

Office COM > WPS COM
```

---

## 5.3 线程模型

* 主线程：UI
* 子线程：转换任务
* 使用线程池（QThreadPool）
